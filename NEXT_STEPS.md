# 逸承工作室下一步建議

## 現在已經有的方向

- 品牌名稱：逸承工作室
- 風格：極簡科技感
- 主案例：Warmth House
- 技術策略：先用 GitHub Pages
- 長期方向：發展成結合科技新知、遊戲與 3C 商城的平台

## 新增重要守則

- 前台不能寫成說明頁或提案頁
- AI 的備註、規劃與提示必須留在 Markdown，不可直接出現在網站畫面
- 單頁文字量要克制，避免滿版長文
- 圖文面積比以 `1.5:1` 為目標，視覺模組要大於純文字
- 整體主風格以未來感為主，電玩遊戲感為輔

詳細規範請以 `DESIGN_GUARDRAILS.md` 為準

## 我建議的網站發展節奏

### 第一階段

先把工作室首頁上線，目標是：

- 建立專業感
- 讓別人知道你在做 AI Coding
- 展示 Warmth House 作為主案例
- 留下聯絡入口

### 第二階段

補成完整工作室官網：

- `about.html`
- `services.html`
- `case-study.html`
- `contact.html`

### 第三階段

開始往科技平台延伸：

- 科技新知分類
- 遊戲內容頁
- 3C 商品整理
- 導購與商城入口

## 上 GitHub Pages 前要補的內容

- 正式聯絡方式
- 社群連結
- 你想公開的自我介紹
- Warmth House 案例更完整的敘述
- 真正的 GitHub repository 名稱

## 這版首頁最值得再調整的地方

- Hero 主標可以再更像你本人說話的語氣
- 主案例可以補 1 到 2 張截圖
- Contact 區塊要換成你的真實資訊
- 如果要更像科技媒體，可以加「內容分類預覽」模組

## 下一步最佳做法

我建議接下來不要急著加很多頁，而是先把首頁重做成真正可公開版本：

- 把提案感文案移出前台
- 增加更強的主視覺與案例截圖
- 把區塊文字再砍短 30% 到 50%
- 用更有未來感的卡片、線條、光感與介面式版型取代說明型排版
- 先把首頁做對，再拆出 `about`、`services`、`case-study`

最合理的執行順序：

1. 重做首頁文案與視覺比例
2. 建立正式案例頁
3. 建立服務頁
4. 最後再補關於頁與聯絡頁
