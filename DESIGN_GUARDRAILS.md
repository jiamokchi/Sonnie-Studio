# 逸承工作室網站設計守則

最後更新：2026-04-07

## 核心原則

這個網站不是 AI 說明頁，也不是提案簡報頁。

它應該是一個真正對外可看的工作室網站：

- 看起來像品牌，而不是文件
- 看起來像作品，而不是備忘錄
- 看起來像未來中的科技工作室，而不是一般接案頁

## 前台內容限制

### 1. 不要把備註寫進前台

任何給 AI、設計、開發或內部討論看的內容，都不能直接出現在使用者第一眼會看到的頁面上。

這類內容應該放在：

- Markdown 文件
- HTML 註解
- 內部備忘錄
- 專案規格文件

前台禁止出現的語氣或內容類型：

- 這一版先怎樣
- 我建議的下一步
- 目前最適合先做的是
- 第一階段 / 第二階段說明口吻太重的內部規劃文
- 像簡報、提案、備忘錄的說法

前台應該只保留：

- 對訪客有價值的品牌訊息
- 精簡服務說明
- 案例亮點
- 明確 CTA

## 版面與內容比例

### 2. 單頁文字不能太多

每個區塊都要避免大段文字堆疊。

頁面原則：

- 一個區塊只講一件事
- 文字短、節奏快、可掃讀
- 圖像、視覺模組、卡片與留白要比文字更有存在感

### 3. 圖文面積比例

整體頁面建議以圖文面積比 `1.5:1` 為目標。

意思是：

- 圖像 / 視覺模組面積要大於純文字區
- 不要做成滿版文字說明頁
- 每個主段落最好都有視覺支點

實作建議：

- Hero 區要有強視覺，不只是一段文案
- 服務區用卡片，不要長文
- 案例區至少放主視覺或網站截圖
- FAQ 保持精簡，避免回答過長

## 視覺方向

### 4. 主風格是未來感

網站整體氣質要優先朝以下方向靠近：

- 未來感
- 科技感
- 俐落
- 冷靜
- 高辨識度

避免：

- 顧問公司感太重
- 簡報感太重
- 溫馨品牌感
- 傳統接案頁感

### 5. 電玩遊戲感是輔助，不是主體

遊戲元素可以作為次要語言點綴，但不能壓過工作室本身的專業感。

可以使用的方式：

- HUD / UI 風格線條
- 像介面面板的卡片
- 科技網格、光暈、掃描感
- 局部像遊戲選單的互動感

不要過度使用：

- 太重的霓虹夜店風
- 太像遊戲官網的誇張特效
- 過度中二的標題與文案

最終感受應該是：

`未來科技工作室 70% + 遊戲介面語言 30%`

## 文案原則

### 6. 文案要像品牌，不像規格書

每段文案都要能被一般訪客快速理解。

避免：

- 過多自我解釋
- 過多流程教學
- 過多內部策略語言

優先保留：

- 你是誰
- 你做什麼
- 你和別人有何不同
- 你做過什麼
- 訪客下一步該做什麼

## 之後改版時必查

每次改版前後，請用這份清單檢查：

- 前台有沒有跑出內部備註語氣
- 單一區塊文字是否過長
- 圖像與卡片面積是否足夠
- 整體氣質是否仍以未來感為主
- 遊戲感是否只是點綴而非主體
- 首頁是否仍然像品牌網站，而不是提案頁
